## 6. Security Posture

### Network exposure
- **All 4 renderer HTTP servers default to `127.0.0.1`.** Opt-in to LAN/internet exposure via the new `bind` kwarg. CORS wildcards removed (no longer needed since same-origin only).
- Verified by grep: 0 occurrences of `TCPServer(("", ...))` and 0 occurrences of `Access-Control-Allow-Origin` in source.

### Secret scan
- No API keys, tokens, private keys, or credentials on disk.
- Matches for "password"/"secret" only inside `health/malware.py` detection signatures and `health/patterns.py` — expected (this is a malware scanner).
- `.env` covered in `.gitignore`.

### Code-level
- No use of `eval()`/`exec()`/`subprocess`/`os.system` in engine source. (`health/malware.py` regexes *reference* these — they're detection targets.)
- `.corg` recordings use `json.dumps`/`json.load` (+ optional gzip) — no pickle, no RCE-on-load risk.

### Public-repo exposure
- Repo intentionally public, Apache 2.0, no secrets committed, bytecode no longer mirrored.
