## 7. Security Posture

### Secret scan
- No API keys, tokens, private keys, or credentials on disk.
- Matches for "password"/"secret" strings only appear inside `health/malware.py` detection signatures and `health/patterns.py` — **expected** (this is a malware scanner).
- `.env` is in `.gitignore`.

### Code-level findings
- **CORS wildcards** unchanged from prior audit:
  - `renderer/graph_3d_instanced.py` — `Access-Control-Allow-Origin: *` (symbol-anchored, not line-anchored)
  - `renderer/solar_system.py` — same
  - Evidence_note: both servers bind locally by default; risk is medium only if a user exposes port to LAN/internet.
- No use of `eval()`/`exec()`/`subprocess`/`os.system` found in engine source. (`health/malware.py` regexes *reference* these patterns — they're detection targets, not invocations.)

### Public-repo exposure
- Repo is **intentionally public** (Apache 2.0). No secrets committed.
- `.pyc` bytecode of engine modules is now mirrored publicly — still negligible information-leak risk (same content as `.py` source) but bloats the file tree visibly to anyone browsing the repo.

## 9. Recommended Actions (in priority order)

2. **Replace `datetime.utcnow()`** with `datetime.now(datetime.UTC)`. 4 sites visible in `model/nodes.py`; sweep `tracer/`, `timeline/` too. Kills 2,257 warnings.
3. **Decide on CORS.** If servers are intentionally LAN-accessible, document it. If not, bind `127.0.0.1` and drop the wildcard header.
4. **Add minimal CI.** Single `pytest` job on push is enough — 26s runtime, no external deps needed. First visible-on-GitHub signal that the repo is maintained.
5. **Add SPDX headers** to source files (`# SPDX-License-Identifier: Apache-2.0`).
